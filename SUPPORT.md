# Support

WarrantOS is maintained on a best-effort, open-source basis with no service-level
agreement. Use GitHub Issues for reproducible defects and feature requests. Include
the WarrantOS version, operating system, Python version, command, exit code and a
minimal redacted fixture. Never attach credentials or protected source material.

Security vulnerabilities must follow `SECURITY.md`, not a public issue. Questions
about whether a WarrantOS result is legally sufficient, factually true or compliant
with a specific regulatory obligation are outside project support.
